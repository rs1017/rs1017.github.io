## Global A4 Rules

- A4 한 장을 뽑는 이유가 분명해야 한다. 한 줄, 한 단어, 한 그림만으로 페이지를 쓰지 않는다.
- 페이지 한 장에는 최소 2개, 가능하면 3개 이상의 학습 행동이 들어가야 한다.
- 활동 행동 예시: 세기, 쓰기, 따라쓰기, 찾기, 연결하기, 색칠하기, 자르기, 붙이기, 비교하기, 고르기.
- 프린터 친화 우선: 흰 배경, 검은 면채움 최소화, 중간 굵기 선, 명확한 여백.
- 시각적 메인과 작업 구역이 같이 있어야 한다. 예: 메인 그림 + tracing + answer choice.
- 페이지마다 난이도 목적을 명시적으로 분리한다. 같은 구조를 숫자만 바꿔 반복하지 않는다.
- 한 게시글의 10장은 서로 다른 페이지 목적 또는 서로 다른 문제 유형을 포함해야 한다.

## Counting Rules

- `1~10개 그림만 나열`하는 페이지를 금지한다.
- 숫자세기 페이지는 다음 중 최소 3개를 같이 넣는다.
  - 개수 세기 메인 장면
  - 숫자 trace
  - 숫자 단어 trace
  - 정답 숫자 고르기
  - ten-frame 채우기
  - count and color
  - more / less 비교
  - missing number 채우기
  - draw more to make N
  - object-to-number matching
- 메인 장면은 `흩어진 오브젝트`가 아니라 `작은 상황 장면`으로 구성한다.
- 오브젝트 수는 1~10이라도, 페이지 목표는 `개수 인식 + 숫자 기호 + 숫자 단어`를 동시에 다룬다.
- 같은 게시글 안에서 10장은 아래처럼 역할을 나눈다.
  - 1~3: 낮은 수량 인식
  - 4~6: count and color / matching
  - 7~8: compare / missing number
  - 9~10: mixed review

## English Rules

- `A4 한 장에 단어 하나만 크게 쓰기` 금지.
- 영어 페이지는 다음 중 최소 3개를 같이 넣는다.
  - 큰 대표 그림
  - target word trace 2~3회
  - uppercase / lowercase letter box
  - 첫 글자 찾기
  - picture-word matching
  - circle the correct word
  - simple cut-and-paste letters
  - beginning sound sort
  - short sentence frame
- 한 페이지는 `단어 소개`가 아니라 `단어 인지 + 문자 형태 + 그림 연결`이 같이 있어야 한다.
- 그림은 허접한 아이콘 1개가 아니라, 큰 대표 그림 + 작은 보조 그림 / 선택지 구조로 간다.
- 단어는 주제 묶음으로 운영한다. 예: farm, fruit, ocean, vehicle.
- 가능하면 파닉스 책처럼 한 페이지에 한 소리(예: A, B, C)만 집중하고, 그 소리로 시작하는 그림 3~4개를 같이 넣는다.
- 파닉스 페이지에서 전혀 관계없는 단어를 섞는 경우는 피한다.
- 같은 게시글 10장은 `단어 소개 / tracing / matching / cut-paste / review`가 섞여야 한다.

## Tracing Rules

- `A4 한 장에 선 1개` 금지.
- 선긋기 페이지는 다음 중 최소 2개 이상을 묶는다.
  - warm-up line strip 3~5개
  - shape tracing block
  - path tracing block
  - finish the picture tracing
  - left-to-right control strip
  - curved / zigzag / loop variation set
- 한 페이지에 6~12개의 trace 동작이 들어가야 한다.
- 위에서 아래로 `쉬운 선 -> 어려운 선 -> 작은 응용 그림` 흐름으로 배치한다.
- tracing은 시각 제어 훈련이므로, 페이지마다 선 종류가 달라야 한다.

## Maze Rules

- 현재 저장소 수준보다 최소 10배 더 복잡하게 만든다.
- 단일 통로형, dead-end 적은 미로 금지.
- A4 한 장이 미로 하나라면 아래 최소 조건을 지킨다.
  - branch decision point 35개 이상
  - dead-end cluster 12개 이상
  - false-route group 3개 이상
  - 시작과 끝이 멀고, 되돌아가기 구간 존재
- 또는 `중형 미로 2개 + 미니 찾기 요소` 조합으로 만든다.
- 단순 선 미로보다 `테마형 배경 + 경로 규칙`이 좋다. 예: collect 3 stars on the way.
- 미로는 쉬운 장식이 아니라 실제로 막히고 돌아가게 만들어야 한다.

## Cut-and-Paste Rules

- `오브젝트 몇 개 잘라 붙이기`보다 `인형 만들기 / 종이 캐릭터 / paper puppet / dress-up doll` 우선.
- 한 페이지에는 최소 다음이 들어간다.
  - base body or puppet base
  - cut pieces 4~8개
  - 붙일 위치 실루엣 또는 가이드
  - 부품 이름 또는 짧은 번호 라벨
- 가능하면 두 페이지 구조로 간다.
  - page 1: 완성 몸체와 붙일 자리
  - page 2: 자를 옷/팔/얼굴/소품 파츠
- `인형 만들기`, `계절 옷 갈아입히기`, `직업 코스튬`, `동물 가면`, `손가락 인형`, `종이 꼭두` 계열을 우선한다.
- 단순 shape sort보다 `완성 결과가 분명한 craft`를 우선한다.

## Hidden Picture Rules

- 기존 저장소의 `빈 장면에 물건 몇 개 숨기기` 방식 금지.
- 앞으로 숨은그림은 `월리를 찾아라`형 dense scene 탐색지로만 만든다.
- 필수 요소:
  - 군중/오브젝트 밀도 높은 큰 장면
  - 정확한 target thumbnail strip
  - 비슷한 distractor 다수
  - 겹침, 가림, 반복 패턴
  - 찾을 대상은 5~12개
- 페이지가 너무 휑하면 실패다.
- 아이가 실제로 한참 찾아야 하는 난도여야 한다.

## Production Note

- 숫자세기, 영어, 선긋기, 오려붙이기, 미로는 `문제 레이아웃`이 핵심이므로 그림 생성과 레이아웃 조립을 분리한다.
- 캐릭터, 오브젝트, 장면 같은 `그림 레이어`는 ComfyUI로 만든다.
- 단어, 문장, tracing 글자, 박스, 번호칸, answer circle, cut line 같은 `레이아웃 레이어`는 별도 조립 단계에서 넣어도 된다.
- 즉, `그림은 ComfyUI`, `텍스트와 학습지 구조는 별도 레이아웃`이 기준이다.
