## 개요

Doc Coauthoring은 문서 공동 작성을 위한 구조화된 워크플로우를 제공하는 스킬입니다. 문서 작성, 제안서, 기술 스펙, 결정 문서 등에 활용됩니다.

## 3단계 워크플로우

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│ 1. 컨텍스트 수집 │───▶│ 2. 구조화 & 정제 │───▶│ 3. 독자 테스트   │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

## Stage 1: 컨텍스트 수집

목표: 사용자가 아는 것과 Claude가 아는 것 사이의 간극 해소

- 문서 유형, 대상 독자, 원하는 영향 파악
- 템플릿이나 특정 형식 확인
- 프로젝트 배경, 조직 컨텍스트 등 정보 수집

## Stage 2: 정제 & 구조화

목표: 섹션별로 브레인스토밍, 큐레이션, 반복 정제

각 섹션마다:
1. 포함할 내용에 대한 명확화 질문
2. 5-20개 옵션 브레인스토밍
3. 사용자가 유지/제거/결합 선택
4. 섹션 초안 작성
5. 외과적 편집으로 반복 정제

## Stage 3: 독자 테스트

목표: 신선한 Claude로 문서 테스트하여 맹점 발견

- 독자가 물을 질문 예측
- 서브에이전트로 테스트 또는 사용자가 수동 테스트
- 모호함, 잘못된 가정, 모순 체크

## 사용 시점

- 문서 작성, 제안서 초안, 스펙 작성 시
- PRD, 설계 문서, 결정 문서, RFC 작성 시
- 상당한 분량의 글쓰기 작업 시작 시
