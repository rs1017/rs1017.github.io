# Mermaid 다이어그램 완벽 가이드

Mermaid는 텍스트 기반으로 다이어그램을 그리는 JavaScript 라이브러리입니다. Markdown 문서에서 코드 블록으로 쉽게 다이어그램을 작성할 수 있습니다.

## 1. 플로우차트 (Flowchart)

### 기본 플로우차트

```mermaid
graph TD
    A[시작] --> B{조건 확인}
    B -->|Yes| C[작업 실행]
    B -->|No| D[종료]
    C --> E[결과 저장]
    E --> D
```

### 방향 설정

```mermaid
graph LR
    A[왼쪽] --> B[오른쪽]
    B --> C[계속]
```

**방향 옵션:**
- `TD` / `TB`: 위에서 아래로 (Top-Down)
- `BT`: 아래에서 위로 (Bottom-Top)
- `LR`: 왼쪽에서 오른쪽으로
- `RL`: 오른쪽에서 왼쪽으로

### 다양한 노드 모양

```mermaid
graph TD
    A[사각형]
    B(둥근 사각형)
    C([스타디움])
    D[[서브루틴]]
    E[(데이터베이스)]
    F((원형))
    G>비대칭]
    H{마름모}
    I{{육각형}}
    J[/평행사변형/]
    K[\역평행사변형\]
```

### 서브그래프

```mermaid
graph TB
    subgraph Frontend
        A[React] --> B[Redux]
        B --> C[Component]
    end
    subgraph Backend
        D[Node.js] --> E[Express]
        E --> F[MongoDB]
    end
    C --> D
```

## 2. 시퀀스 다이어그램 (Sequence Diagram)

### 기본 시퀀스

```mermaid
sequenceDiagram
    participant Client
    participant Server
    participant Database

    Client->>Server: HTTP 요청
    Server->>Database: 쿼리 실행
    Database-->>Server: 결과 반환
    Server-->>Client: HTTP 응답
```

### 활성화와 노트

```mermaid
sequenceDiagram
    participant U as 사용자
    participant A as 인증 서버
    participant S as 서비스

    U->>+A: 로그인 요청
    Note right of A: 자격증명 검증
    A->>A: 비밀번호 확인
    A-->>-U: JWT 토큰 반환

    U->>+S: API 요청 (토큰 포함)
    Note over U,S: 토큰 검증 과정
    S-->>-U: 데이터 반환
```

### 루프와 조건문

```mermaid
sequenceDiagram
    participant C as Client
    participant S as Server

    C->>S: 연결 요청

    loop 매 5초마다
        S->>C: 하트비트 전송
        C->>S: 하트비트 응답
    end

    alt 연결 성공
        S->>C: 데이터 스트림 시작
    else 연결 실패
        S->>C: 오류 메시지
    end
```

## 3. 클래스 다이어그램 (Class Diagram)

### 기본 클래스

```mermaid
classDiagram
    class Animal {
        +String name
        +int age
        +makeSound()
        +move()
    }

    class Dog {
        +String breed
        +bark()
        +fetch()
    }

    class Cat {
        +String color
        +meow()
        +scratch()
    }

    Animal <|-- Dog
    Animal <|-- Cat
```

### 관계 표현

```mermaid
classDiagram
    class User {
        +int id
        +String name
        +login()
    }

    class Order {
        +int orderId
        +Date orderDate
        +process()
    }

    class Product {
        +int productId
        +String name
        +float price
    }

    User "1" --> "*" Order : places
    Order "*" --> "*" Product : contains
```

## 4. 상태 다이어그램 (State Diagram)

### 주문 상태

```mermaid
stateDiagram-v2
    [*] --> 주문접수
    주문접수 --> 결제대기: 주문 확인
    결제대기 --> 결제완료: 결제 성공
    결제대기 --> 주문취소: 결제 실패
    결제완료 --> 배송준비: 재고 확인
    배송준비 --> 배송중: 출고
    배송중 --> 배송완료: 도착
    배송완료 --> [*]
    주문취소 --> [*]
```

### 중첩 상태

```mermaid
stateDiagram-v2
    [*] --> Active

    state Active {
        [*] --> Idle
        Idle --> Processing: 작업 시작
        Processing --> Idle: 작업 완료
    }

    Active --> Inactive: 비활성화
    Inactive --> Active: 활성화
    Inactive --> [*]
```

## 5. ER 다이어그램 (Entity Relationship)

```mermaid
erDiagram
    USER ||--o{ ORDER : places
    USER {
        int id PK
        string name
        string email
        date created_at
    }

    ORDER ||--|{ ORDER_ITEM : contains
    ORDER {
        int id PK
        int user_id FK
        date order_date
        string status
    }

    ORDER_ITEM }|--|| PRODUCT : references
    ORDER_ITEM {
        int id PK
        int order_id FK
        int product_id FK
        int quantity
    }

    PRODUCT {
        int id PK
        string name
        float price
        int stock
    }
```

## 6. 간트 차트 (Gantt Chart)

```mermaid
gantt
    title 프로젝트 일정
    dateFormat  YYYY-MM-DD

    section 기획
    요구사항 분석      :done,    req, 2024-01-01, 7d
    기술 검토          :done,    tech, after req, 5d

    section 개발
    백엔드 개발        :active,  back, 2024-01-13, 14d
    프론트엔드 개발    :         front, 2024-01-15, 14d
    API 연동          :         api, after back, 7d

    section 테스트
    단위 테스트        :         unit, after front, 5d
    통합 테스트        :         integ, after api, 5d

    section 배포
    스테이징 배포      :         staging, after integ, 3d
    프로덕션 배포      :milestone, prod, after staging, 1d
```
